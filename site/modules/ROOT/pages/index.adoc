= Neo4j + AWS Bedrock: GraphRAG Workshop

== Welcome

This workshop teaches you to build Graph Retrieval-Augmented Generation (GraphRAG) pipelines using Neo4j Aura and Amazon Bedrock. You will work with SEC 10-K financial filing data — companies, products, risk factors, and institutional ownership — progressing from no-code tools to building custom AI agents.

== Graph Intelligence Meets AWS AI

Neo4j Aura is the Graph Intelligence Platform. It makes connections between entities explicit and traversable. Amazon Bedrock provides managed access to foundation models for generation and embedding. Together they form a complete GraphRAG stack: Neo4j handles topology and retrieval, Bedrock handles reasoning and generation.

The integration covers four patterns: vector search over embedded chunks stored in Neo4j, graph-enriched retrieval that follows relationships from matched chunks to connected entities, text-to-Cypher generation that translates natural language into graph queries, and agent orchestration that routes questions to the right retrieval strategy.

=== Neo4j Aura: Topology and Retrieval

Neo4j stores the SEC financial knowledge graph — companies, their products, risk factors, competitors, partners, and institutional owners — as nodes and relationships. Built-in vector indexes store embeddings alongside graph data, enabling semantic similarity search without a separate vector store. Cypher queries combine vector search with graph traversal in a single operation. See xref:lab1.adoc[Lab 1] for graph setup and Cypher basics.

=== Amazon Bedrock: Reasoning and Generation

Amazon Bedrock provides the AI backbone. Claude models handle reasoning, tool selection, and response generation. Amazon Nova Multimodal Embeddings generate the vectors stored in Neo4j's vector indexes. LangGraph and the Strands SDK provide agent orchestration frameworks that connect to Neo4j through MCP (Model Context Protocol).

== From Knowledge Graph to Agents

Three stages build on each other across the workshop.

**Knowledge Graph Construction.** CSV seed data loads into Neo4j as nodes and relationships. Unstructured 10-K filing text is chunked, embedded with Nova, and linked to extracted entities. The result is a searchable knowledge graph with both structured and unstructured data.

**GraphRAG Retrieval.** Vector search finds semantically similar chunks. Graph traversal enriches results with connected entities — the company that filed the document, related products, shared risk factors. Fulltext search handles keyword queries. Hybrid approaches combine all three.

**Agent Orchestration.** LangGraph agents use the ReAct pattern to select retrieval tools, execute queries, and synthesize answers. The Neo4j MCP Server exposes schema inspection and read-only Cypher as agent tools. Aura Agents provides a no-code path to the same capabilities.

== Knowledge Graph Schema

The SEC financial dataset includes six entity types connected by typed relationships:

----
(:Company)       -[:OFFERS]->         (:Product)
(:Company)       -[:FACES_RISK]->     (:RiskFactor)
(:Company)       -[:COMPETES_WITH]->  (:Company)
(:Company)       -[:PARTNERS_WITH]->  (:Company)
(:AssetManager)  -[:OWNS {shares}]->  (:Company)
(:Company)       -[:FILED]-> (:Document) <-[:FROM_DOCUMENT]- (:Chunk)
----
