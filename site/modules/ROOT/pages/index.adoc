= Neo4j + AWS Bedrock: GraphRAG Workshop

== Welcome

This workshop teaches you to build Graph Retrieval-Augmented Generation (GraphRAG) pipelines using Neo4j Aura and Amazon Bedrock. You will work with SEC 10-K financial filing data — companies, products, risk factors, and institutional ownership — progressing from no-code tools to building custom AI agents.

== Graph Intelligence Meets Generative AI on AWS

Neo4j Aura is the Graph Intelligence Platform. It makes connections between entities explicit and traversable. Amazon Bedrock provides managed access to foundation models for generation and embedding. Together they form a complete GraphRAG stack: Neo4j handles topology and retrieval, Bedrock handles reasoning and generation.

The integration covers four patterns: vector search over embedded chunks stored in Neo4j, graph-enriched retrieval that follows relationships from matched chunks to connected entities, text-to-Cypher generation that translates natural language into graph queries, and agent orchestration that routes questions to the right retrieval strategy.

=== Neo4j: Graph Intelligence Platform

Neo4j is the world's leading Graph Intelligence Platform — AI-ready data by design. Neo4j AuraDB, the fully managed graph database as a service, stores data as nodes and relationships with built-in vector indexes that keep embeddings alongside graph data. This design enables GraphRAG: retrieval that combines semantic similarity search with graph traversal in a single query. Neo4j Aura Agent provides a no-code path to build, test, and deploy AI agents directly from AuraDB knowledge graphs. In this workshop, Neo4j stores the SEC financial knowledge graph and serves as the retrieval layer for all agent queries. See xref:lab1.adoc[Lab 1] for graph setup and Cypher basics.

=== Generative AI on AWS

AWS provides the most comprehensive set of generative AI tools and services for building applications at production scale. Amazon Bedrock is the end-to-end platform for building generative AI applications and agents, providing unified API access to foundation models — Anthropic Claude for reasoning and generation, Amazon Nova for embeddings. Amazon Bedrock AgentCore is the agentic platform for building, deploying, and operating agents securely at scale without infrastructure management. Amazon SageMaker AI is the managed service for building, training, and deploying ML and foundation models, including the JupyterLab notebook environment used throughout this workshop. See xref:aws-services.adoc[AWS GenAI Services] for detailed coverage of each service.

== From Knowledge Graph to Agents

Three stages build on each other across the workshop.

**Knowledge Graph Construction.** CSV seed data loads into Neo4j as nodes and relationships, forming the structured layer of companies, products, risk factors, and institutional ownership. Unstructured 10-K filing text is chunked and embedded with Nova, then linked to the structured layer through Document nodes. The result is a searchable knowledge graph that connects semantic text to typed entities.

**GraphRAG Retrieval.** Vector search finds semantically similar chunks. Graph traversal enriches results with connected entities — the company that filed the document, related products, shared risk factors. Fulltext search handles keyword queries. Hybrid approaches combine all three.

**Agent Orchestration.** AI agents use the ReAct pattern to select retrieval tools, execute queries, and synthesize answers. The Strands Agents SDK provides the agent framework, and the Neo4j MCP Server exposes schema inspection and read-only Cypher as agent tools. Aura Agents provides a no-code path to the same capabilities.

== The SEC 10-K Dataset

The workshop uses data from SEC 10-K filings, the annual reports that publicly traded companies file with the U.S. Securities and Exchange Commission. Each filing contains both structured data (company details, financial metrics, institutional ownership) and unstructured narrative text (risk factor discussions, business descriptions, executive commentary). That combination makes 10-K filings a natural fit for GraphRAG: the structured data forms a queryable graph of entities and relationships, while the narrative text provides the semantic content that embedding-based retrieval searches over.

The dataset covers approximately 76 companies across technology, finance, healthcare, and automotive sectors. It includes 274 products, 203 risk factors, and 15 institutional asset managers with their portfolio holdings. Questions like "Which risk factors expose an asset manager's portfolio across multiple sectors?" require traversing structured relationships. Questions like "What do companies say about supply chain disruption?" require searching unstructured filing text by meaning. GraphRAG handles both, and the combination of both in a single query is where the architecture earns its complexity.
