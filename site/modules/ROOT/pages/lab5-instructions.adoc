= Lab 5 Instructions: Neo4j MCP Server

TIP: For concepts and background, see xref:lab5.adoc[Lab 5: Neo4j MCP Server].

== What You'll Learn

* **MCP Fundamentals**: Connect a Strands Agent to a Neo4j MCP Server, discover tools, inspect the graph schema
* **Cypher Templates**: Pre-written vector search + graph traversal queries in `@tool` functions for reliable, predictable results
* **Text2Cypher**: The agent discovers the schema and writes original Cypher from scratch — fully autonomous graph querying

== Prerequisites

* Completed **Lab 1** (Neo4j Aura instance with SEC financial data loaded)
* Your `CONFIG.txt` file updated with MCP Gateway credentials (`MCP_GATEWAY_URL` and `MCP_ACCESS_TOKEN`)
* AWS credentials configured for Amazon Bedrock access

NOTE: The MCP server is pre-configured by the lab administrator with full embeddings and indexes. You do not need to complete Lab 4 before starting this lab.

== Notebooks

Open each notebook in order in the `Lab_5_MCP_Server/` directory. The progression builds from MCP basics to autonomous agents.

[cols="1,2,3"]
|===
| Notebook | Title | What You Build

| `01_intro_strands_mcp.ipynb`
| Intro to Strands + MCP
| MCP connection via Streamable HTTP, tool discovery with `list_tools_sync()`, graph schema inspection

| `02_graph_enriched_search.ipynb`
| Graph-Enriched Search
| Cypher Templates — pre-written `@tool` functions that combine vector search with graph traversal for document context and entity enrichment

| `03_text2cypher_agent.ipynb`
| Text2Cypher Agent
| An autonomous agent that discovers the graph schema and writes original Cypher from scratch — the Text2Cypher retrieval pattern
|===
