= Lab 4 Instructions: Graph-Enriched Search

Follow these notebooks to search a Neo4j knowledge graph, progressing from vector search to graph-enriched retrieval to hybrid search.

TIP: For concepts and background, see xref:lab4.adoc[Lab 4: Graph-Enriched Search].

== What You'll Learn

* **Vector Search via MCP**: Embed queries and execute semantic search through an MCP agent
* **Graph-Enriched Retrieval**: Follow relationships from matched chunks to documents, companies, and risk factors
* **Hybrid Search**: Combine fulltext keyword search with vector similarity using custom `@tool` wrappers

== Prerequisites

* Completed **Lab 1** (Neo4j Aura instance created and running)
* `CONFIG.txt` at the project root filled in with Neo4j credentials, `MCP_GATEWAY_URL`, and `MCP_ACCESS_TOKEN`
* A running environment (SageMaker Studio or GitHub Codespace)

NOTE: The Neo4j MCP Server used in this lab connects to a database that has been pre-loaded with Document and Chunk nodes, vector embeddings, and fulltext indexes. The workshop admin set this up before the workshop using the data loading pipeline. You do not need to load this data yourself — the MCP Server's database is ready to query.
