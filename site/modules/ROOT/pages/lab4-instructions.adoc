= Lab 4 Instructions: GraphRAG Retrievers

Follow these notebooks to load chunk embeddings into your knowledge graph, build retrieval pipelines that progress from pure vector search to graph-enriched retrieval, then combine both retrievers in a Strands agent that chooses the right strategy per question.

TIP: For concepts and background, see xref:lab4.adoc[Lab 4: GraphRAG Retrievers].

== What You'll Learn

* **Data Loading**: Add document chunks with pre-computed embeddings to the existing knowledge graph from Lab 1
* **Vector Search**: Use `VectorRetriever` to find semantically similar chunks and generate answers with `GraphRAG`
* **Graph-Enriched Search**: Use `VectorCypherRetriever` to combine vector search with Cypher graph traversal for richer context
* **Agent-Driven Retrieval**: Wrap both retrievers as Strands `@tool` functions and let the agent choose the right strategy per question

== Prerequisites

* Completed **Lab 1**: Aura instance created and knowledge graph loaded
* Completed **Lab 3**: SageMaker Studio set up and repository cloned
* A running environment (SageMaker Studio or GitHub Codespace)

NOTE: This lab uses the same Neo4j Aura instance from Labs 1-2. The structured graph (companies, products, risk factors, asset managers) must already be loaded. Notebook 01 adds the unstructured layer on top; it does not clear existing data.

=== Update CONFIG.txt with Neo4j Credentials

Before running the notebooks, open `CONFIG.txt` at the repository root and fill in the Neo4j credentials you saved in Lab 1:

[source,text]
----
NEO4J_URI=neo4j+s://xxxxxxxx.databases.neo4j.io
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=your-password-from-lab-1
----

`MODEL_ID` and `REGION` are pre-configured with working defaults. See xref:configuration.adoc[Configuration Reference] for details on all fields.

== Notebooks

[cols="1,2,3"]
|===
| Notebook | Title | What You Build

| `01_load_and_query.ipynb`
| Load Data and Query
| Load chunk embeddings into the graph, create a vector index, link entities to chunks with `FROM_CHUNK` relationships, and run test queries

| `02_vector_retriever.ipynb`
| Vector Retriever
| Semantic search with `VectorRetriever` and end-to-end question answering with `GraphRAG`

| `03_vector_cypher_retriever.ipynb`
| VectorCypher Retriever
| Graph-enriched retrieval using `VectorCypherRetriever`, adding filing metadata, companies, products, and risk factors to vector search results

| `04_strands_graphrag_agent.ipynb`
| Strands GraphRAG Agent
| A Strands agent with both retrievers as `@tool` functions that decides which retrieval strategy to use based on the question
|===
