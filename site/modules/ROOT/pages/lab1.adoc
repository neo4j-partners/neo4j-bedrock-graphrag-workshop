== Graph Databases

A graph database models data as nodes and relationships. Nodes represent entities. Relationships represent the connections between them. Both carry properties, which are key-value pairs that describe their attributes.

The notation follows a visual convention. Parentheses denote nodes, brackets denote relationships:

----
(:Company {name, ticker})-[:FACES_RISK]->(:RiskFactor)
----

Each Company node carries properties like name and ticker. Each FACES_RISK relationship carries details like severity. The relationship is directional, typed, and stored as a first-class element alongside the nodes it connects.

.How graphs compare to relational databases
[%collapsible]
====
Where relational databases represent connections implicitly through foreign keys and join tables, graphs make them explicit and traversable. Questions that require following chains of connections become single traversal patterns rather than nested JOINs.
====

Cypher is the query language for interacting with Neo4j. It uses a pattern-matching syntax that mirrors the visual structure of nodes and relationships:

[source,cypher]
----
MATCH (c:Company {ticker: 'NVDA'})-[:OFFERS]->(p:Product)
RETURN p.name ORDER BY p.name LIMIT 10
----

.How the query works
[%collapsible]
====
The query reads like the pattern it matches: start at a Company node with ticker NVDA, follow OFFERS relationships to Product nodes, and return the product names. Cypher handles multi-hop traversals and path-finding operations natively.
====

== The SEC Financial Knowledge Graph

The workshop builds a knowledge graph from SEC 10-K financial filings. Four core entity types and the relationships between them capture the structure of public companies, their products, risk factors, and institutional ownership:

[cols="1,3"]
|===
| Entity Type | Examples

| **Company**
| NVIDIA, Microsoft, Apple, Amazon, Tesla, JPMorgan Chase, Goldman Sachs, Johnson & Johnson, Pfizer

| **Product**
| GPUs, Cloud Services, iPhone, AWS, Model S, Investment Banking

| **RiskFactor**
| Supply Chain Disruption, Regulatory Compliance, Cybersecurity Threats

| **AssetManager**
| BlackRock, Vanguard, State Street, Fidelity
|===

These entities connect through typed relationships that reflect real-world structure:

----
(Company)-[:OFFERS]->(Product)
(Company)-[:FACES_RISK]->(RiskFactor)
(Company)-[:COMPETES_WITH]->(Company)
(Company)-[:PARTNERS_WITH]->(Company)
(AssetManager)-[:OWNS {shares}]->(Company)
----

A question like "Which risk factors expose an asset manager's portfolio across multiple companies?" requires traversing from AssetManager through OWNS to Company, then FACES_RISK to RiskFactor, grouping by risk factor. That multi-hop traversal is what the graph handles natively.

== Neo4j Aura

Neo4j Aura is a fully managed cloud graph database service available through the AWS Marketplace. It provides the Neo4j graph engine without the operational overhead: automatic backups, high availability, and deployment on AWS. Each participant provisions their own Aura instance to hold the SEC financial knowledge graph.

.Vector search, Graph Data Science, and AI capabilities
[%collapsible]
====
Aura provides capabilities beyond the core graph engine that support AI retrieval pipelines. Built-in vector indexes store embeddings alongside graph data, enabling semantic similarity search without a separate vector store. Cypher queries can combine vector search with graph traversal in a single operation. Graph Data Science algorithms like PageRank and community detection run directly within Aura. These capabilities come together in Lab 6 (GraphRAG retrieval) and Lab 7 (MCP agents).
====

.Aura Agents — no-code agent platform
[%collapsible]
====
Aura Agents is Neo4j's no-code agent platform for building conversational interfaces grounded in graph data. Users describe a use case, and the platform auto-generates an agent with tools tailored to the graph's schema: Cypher query templates for deterministic traversals, vector similarity search over embedded content, and text-to-Cypher generation for ad-hoc exploration. Agents deploy as REST API endpoints or MCP server endpoints. See xref:lab1-explore.adoc[Explore the Graph] for visual exploration, and Lab 2 for the full Aura Agents workflow.
====

== Aura Developer Tools

The Aura console provides three tools you will use throughout the workshop.

**Query** is a developer environment for writing and executing Cypher queries with syntax highlighting, auto-completion, and saved query collections. You will use this to load data and verify results.

**Explore** (powered by Neo4j Bloom) provides visual graph exploration through an interactive canvas. Search for nodes by label or property, expand their relationships, and discover patterns visually.

**Dashboards** offer low-code data visualization including bar charts, geographic maps, and graph visualizations for presenting graph data to non-technical stakeholders.
